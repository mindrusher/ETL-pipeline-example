# End-to-End Data Engineering Pipeline (Airflow + Postgres)

## Обзор

Этот проект реализует полный комплексный pipeline обработки данных с использованием Apache Airflow, PostgreSQL и Docker. Он демонстрирует основные концепции обработки данных, включая инкрементальную загрузку, хранилища данных, оркестрацию и автоматизацию резервного копирования.
Pipeline извлекает данные из OLTP-системы, обрабатывает их на нескольких уровнях и предоставляет доступ к ним для инструментов бизнес-аналитики, таких как Metabase.


## Архитектура

Pipeline обработки данных имеет многоуровневую архитектуру:


OLTP → MRR (Raw Layer) → STG (Staging) → DWH (Core) → BI
                 ↓
         High Water Mark + Logging


## Подробнее о слоях

### 1. OLTP (исходные данные)

* Имитирует транзакционную базу данных
* Содержит таблицы: клиенты, товары, продажи
* Включает столбец updated_at для инкрементального извлечения данных


### 2. MRR (сырые данные)

* Сохраняет данные, извлеченные из OLTP
* Реализует инкрементальную загрузку с использованием High Water Mark
* Использует UPSERT (при конфликте) для обеспечения идемпотентности


### 3. STG (слой предварительной загрузки)

* Временный слой преобразования
* Данные полностью перезагружаются с помощью команды `TRUNCATE + INSERT`
* Используется для очистки и подготовки данных

> Примечание: Для простоты используется полная перезагрузка.


### 4. DWH (централизованная база)

* Реализация звезды:
  * dim_customers
  * dim_products
  * fact_sales
* Таблица фактов использует инкрементальную загрузку с использованием High Water Mark
* Уникальные ограничения обеспечивают идемпотентные вставки


### 5. BI слой

* Данные предоставляются через SQL VIEW
* Возможно подключение к Metabase


## Технологии

* Apache Airflow (orchestration)
* PostgreSQL (databases)
* Docker & Docker Compose
* Python (ETL logic)
* psycopg2 (DB connection)


## Поток данных

1. **OLTP → MRR**

   * Пошаговое извлечение с использованием updated_at
   * Отслеживание последней даты обновления (High Water Mark)
   * UPSERT логика

2. **MRR → STG**

   * Полная перезагрузка (truncate + insert)

3. **STG → DWH**

   * Измерения: вставка без дубликатов
   * Факт: инкрементальная загрузка с использованием временной метки

4. **Backup**

  * Автоматизировано с помощью pg_dump
  * Выполняется как заключительная задача Airflow

## Установка и запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/mindrusher/ETL-pipeline-example.git

# 2. Запустить все сервисы из папки с проектом
docker-compose up -d

установить Faker внутри контейнеров airflow (pip install Faker) для реалистичных тестовыъ данных

# 3. Для инициализации бд, запустить DAG: init_db_dag
# 4. Для запуска пайплайна запустить DAG: main_etl_dag
```
